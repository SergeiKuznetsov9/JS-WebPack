Теперь в случае изменения кода нам всякий раз нужно перебилживать проект, чтобы увидеть изменения.
Это не удобно, поэтому мы это исправим и сделаем два разных конфига вебпака - дев и прод.

Для начала установим webpack-dev-server:
npm install --save-dev webpack-dev-server

далее посоздаем фалый конфигов вебпака

После этого команда npx webpack serve --config webpack.dev.config.js сбилдит и запустит проект
Теперь все работает.

Однако вебпак сейчас распознает только js файлы. Мы конечно можем добавить напрямую css но тогда мы
потеряем в оптимизации.

Webpack по умолчанию понимает только файлы javascript и json. Чтобы это изменить нужно использовать
loaders. Они добавляют возможность загружать и обрабатывать другие типы файлов, рассматривая каждый
из них почти как другой модуль javascript. Это позволяет вебПаку добавлять их в свой граф
зависимостей и оптимизировать сборку.

Как и в случае с плагинами, существует широкий набор загрузчиков для различных типов файлов
и функциональности. Необходимо установить загрузчик, чем-то похожий на плагин, а затем настроить
его в конфигурациях вебПака.
